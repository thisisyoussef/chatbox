# SC-008D Design Brief

## Metadata

- Story ID: SC-008D
- Story Title: Partner submission intake, review queue, and catalog hydration
- Author: Codex
- Date: 2026-04-05

## Intent

Close the last major partner-DX gap by turning reviewed app onboarding from a
docs-only contract into a host-owned product flow inside Chatbox.

## Scope

- settings-visible developer and reviewer surface
- manifest validation against the existing reviewed-app contract
- optional single-file HTML runtime upload
- persisted submission queue
- approve or reject controls
- catalog hydration for approved apps
- runtime package lookup when approved apps launch

## Non-Goals

- external marketplace backend
- organization-level reviewer permissions
- multi-file package bundling
- remote deployment of third-party runtimes
